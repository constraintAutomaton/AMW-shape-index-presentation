Web Querying and Decentralized Environments: The Motivation

  • The World Wide Web as a database
    • A wealth of information (regardless of quality), from personal to entertainment, scientific, political, and so ...
    • Systems can exploit this information for different kind of applications from social media, statistical analysis, games and so on...
  • Query language webSQL (Querying the World Wide Web [1996], Mendelzon et al.)
  • Rely on indexing for dataset discovery, and data might not be machine understandable

Linked data and Decentralized information: The Context

  • Web data as an RDF knowledge graph in the form of triples (subject, predicate, object)
  • Conjective queries using SPARQL
  • Data can be described using IRI
  • Dereferencing IRI provided more information about a term
  • Possibilities for data discovery with no indexing via IRI dereferencing
  • Possibilities of descentralization without indexes with machine interpretable data that can be reason upon (we focus on the querying in this presentation)

Link Traversal Query Processing: What it is and the Challenges

  • Link Traversal Query Processing (LTQP) (Foundations of traversal based query execution over linked data [2012], Hartig et al.)
  • Starting from a set of IRI
  • Recursive dereferencing of IRI from an internal query store
  • Dereferencing operations based on a policy (derived from reachability criteria)
  • Two main challenges
    • Pseudo infinite search space => reachability criteria limit the space
    • Lack of a priori information for query planning => Heuristic based query planning (Zero-Knowledge Query Planning for an Iterator Implementation of Link Traversal Based Query Execution [2011], Hartig)

Structured Linked Data Environments: A new context for Traversal Querying?

  • Most research on LTQP has focus on the open-web
  • Data publication can be structured using structural assumption (Link traversal query processing over decentralized environments with structural assumptions [2023], Taelman and Verborgh)
  • Structural assumption can be used for Query optimization by reducing query engine search space
  • Could be used for query planing

Conclusion

  • Shape indexes don't require a lot of maintenance from the data publisher (only on updates that modify the data model or the location of data)
  • Could improve data quality in the publication of decentralized RDF dataset
  • Questions left to be answered
    • What is the complexity/overhead of the containment algorithm with regards to different queries and shape profiles
    • Does the size of the internal triple store have more impact than the reduction in the number of HTTP requests
    • What is the performance in a more uncertain environment: open/closed shapes, complete/incomplete shape index, not every dataset has a shape index, more or less descriptive shapes
    • Can the shape index be used for query planning
